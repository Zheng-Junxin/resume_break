# 构建计划书：Resume Break Microsoft Edge 扩展

## 1. 产品目标

构建一个面向求职者的 Microsoft Edge 扩展，在浏览岗位 JD 网页时抓取岗位描述，结合用户已上传并解析的简历内容，输出岗位匹配建议，并生成可编辑、可换模板、可导出的岗位专属定制简历。

## 2. 核心用户流程

1. 用户打开招聘 JD 网页。
2. 点击浏览器扩展按钮，抓取当前页面 JD。
3. 扩展打开简历工作台。
4. 用户上传或粘贴简历，系统解析为结构化模块。
5. 用户校正个人信息、经历、项目、技能、教育、证书等内容。
6. 用户可编辑模块标题、调整模块顺序、添加自定义模块。
7. 系统根据 JD 关键词生成优化建议和定制简历。
8. 用户选择简历模板，预览并导出 PDF 或 Word。

## 3. 当前实现范围

- 插件类型：Microsoft Edge Manifest V3 扩展。
- 页面能力：通过 `activeTab` 和 `scripting` 在用户触发后读取当前网页文本。
- 数据存储：使用 `chrome.storage.local`，不上传用户简历和 JD。
- 简历解析：支持 TXT、Markdown、HTML、DOC、DOCX、PDF；PDF 使用扩展内置 PDF.js 本地解析。
- 编辑能力：支持头像上传、模块顺序调整、模块标题编辑、自定义模块添加。
- 定制逻辑：本地关键词抽取、匹配、缺口提示、经历和技能优先级重排。
- LLM 能力：支持 OpenAI-compatible `base_url + /chat/completions`，用于模块格式整理、JD 解析和岗位定制简历生成。
- 模板能力：内置专业经典、现代强调、紧凑 ATS 三套预览/导出模板。
- 导出能力：浏览器打印保存 PDF；生成 Word 兼容 `.doc` 文件。

## 4. 信息架构

- `manifest.json`：Edge 扩展清单。
- `popup.html`：扩展按钮弹窗，用于抓取本页 JD 和打开工作台。
- `app.html`：完整简历定制工作台。
- `export.html`：导出预览页。
- `src/content.js`：招聘网页 JD 抽取脚本。
- `src/resume-core.js`：简历解析、JD 分析、定制、模板渲染、导出核心逻辑。
- `src/app.js`：工作台交互逻辑。
- `src/popup.js`：弹窗交互逻辑。
- `src/export.js`：导出页交互逻辑。
- `styles/`：弹窗、工作台、导出页样式。
- `lib/pdfjs/`：内置 PDF.js，用于本地 PDF 简历解析。

## 5. 简历模板结构

统一内容结构包含：

- 个人信息
- 个人评价
- 专业技能
- 实习 / 工作经历
- 项目经历
- 教育经历
- 荣誉 / 证书
- 用户自定义模块

当前模板：

- 专业经典：正式、单栏、适合通用岗位申请。
- 现代强调：视觉层次更强，适合作品集、产品、运营和复合型岗位。
- 紧凑 ATS：信息密度更高，适合经历较多或需要机器解析的场景。

## 6. 匹配与优化策略

- 从 JD 中抽取技术、业务、岗位能力关键词。
- 在简历各模块中检测关键词覆盖情况。
- 将与 JD 匹配度更高的经历和技能前置。
- 对缺失但 JD 高频出现的关键词给出补充建议。
- 对项目和经历提示量化结果补充，例如规模、性能、转化率、准确率、成本节省。
- 不凭空生成未经用户确认的经历。

## 7. 安全与隐私设计

- 默认只在用户点击扩展按钮后读取当前标签页。
- 不使用 `<all_urls>` 常驻内容脚本。
- 本地功能不调用外部接口。
- LLM 相关功能仅在用户主动点击时调用配置的模型服务。
- 不在代码中内置密钥。
- API Key 不写入代码，仅保存在浏览器本地扩展存储。
- 文件上传限制为 8MB 内的简历文件。
- 所有导出均在本地浏览器完成。

## 8. 后续可扩展方向

- 增加更多简历模板和模板参数配置。
- 增加扫描件 PDF 的 OCR 流程。
- 增加 ATS 评分报告。
- 增加 OneDrive / Microsoft Graph 保存能力。
- 增加 Word Add-in 版本，支持在 Word 内直接编辑当前简历。
