# JD Resume Tailor

一个项目内实现的 Microsoft Edge Manifest V3 扩展，用于在招聘 JD 页面抓取岗位描述，并基于用户简历生成岗位专属定制简历。

## 功能

- 在招聘网页中抓取 JD 正文和页面标题。
- 上传或粘贴简历文本，解析为个人信息、个人评价、专业技能、实习/工作经历、项目经历、教育经历、荣誉/证书。
- 支持 TXT、Markdown、HTML、DOC、DOCX、PDF 简历解析。
- 支持头像上传，并在 PDF / Word 模板中展示。
- 结构化模块支持顺序调整。
- 根据 JD 关键词重排经历和技能，生成优化建议与定制稿。
- 可配置 OpenAI-compatible 模型服务的 `base_url`、`model`、`api key`。
- 结构化编辑支持 LLM 单模块格式整理和全简历统一格式。
- JD 匹配支持 LLM 仅解析 JD，定制简历支持 LLM 生成定制稿。
- 自动搜索 / 自动投递：支持搜索 URL 模板、关键词筛选、排除词、匹配分阈值、投递上限、多步填表、站内已保存简历入口、可选自动点击最终投递。
- 自动投递日志：记录搜索、跳过、填表、提交、需要手动上传等状态。
- 使用统一专业简历模板预览。
- 导出 PDF：打开导出页后使用浏览器打印保存为 PDF。
- 导出 Word：下载 Word 兼容 `.doc` 文档。

## 本地安装

1. 打开 Microsoft Edge。
2. 进入 `edge://extensions/`。
3. 打开“开发人员模式”。
4. 点击“加载解压缩的扩展”。
5. 选择本目录：`D:\A_Hunterstools\resume_break`。

## 使用流程

1. 打开一个招聘 JD 网页。
2. 点击 Edge 工具栏中的 `JD Resume Tailor`。
3. 点击“抓取本页 JD 并打开工作台”。
4. 在工作台上传 TXT / Markdown / HTML / DOC / DOCX / PDF 简历，或直接粘贴简历文本。
5. 点击“解析简历”，在结构化编辑区修正内容。
6. 如需使用模型，在“模型设置”中填写 Base URL、Model、API Key。
7. 点击“本地定制”或“LLM 定制”。
8. 根据建议补充真实经历，最后导出 PDF 或 Word。

## 自动搜索 / 自动投递

1. 在“自动搜索 / 自动投递”中填写关键词、地点、搜索 URL 模板。
2. 搜索 URL 模板支持 `{keywords}` 和 `{location}` 占位符。
3. 填写“必须包含”和“排除关键词”，设置最低匹配分、最多投递、最多职位页。
4. 填写姓名、邮箱、手机、作品集等自动填表信息。
5. 如需真正自动提交，勾选“允许自动点击最终投递 / Submit”。
6. 点击“开始自动搜索”。

默认不会自动点击最终提交按钮，只会自动打开页面、抓取 JD、评分、填表、点击下一步并停在确认步骤。开启自动提交后，扩展会在没有登录墙、附件上传和未识别必填项时尝试点击页面上的提交/投递按钮。

## 隐私与边界

- 本地解析和本地定制不调用外部服务。
- 只有点击 LLM 按钮时，才会把相关 JD / 简历内容发送到用户配置的模型服务商。
- 数据保存在浏览器 `chrome.storage.local`。
- API Key 也保存在浏览器本地；如果使用共享电脑，请及时清空扩展数据。
- 弹窗抓取默认使用 `activeTab`，只读取用户当前点击扩展时所在页面。
- 工作台内再次抓取 JD 时，浏览器会按需请求对应招聘网站的访问授权。
- 自动搜索/投递会在开始时请求网页访问授权；这是为了能跨搜索结果自动打开并处理不同招聘网站。
- 自动投递遇到登录、验证码、未识别必填项或网站要求手动确认时，会停留在对应页面。
- 浏览器安全策略不允许脚本自动给文件上传控件设置本地文件；如果招聘网站要求上传附件简历，扩展会停在该页面并提示手动上传。
- 不同招聘网站的结构、反自动化策略和服务条款不同，自动投递可能需要你按网站规则调整搜索模板和筛选条件。
- PDF 解析使用扩展内置的 PDF.js 本地库；扫描件 PDF 仍需要先 OCR。
- DOCX 解析依赖现代 Edge 的本地解压能力；如果文件结构特殊，请复制文本后粘贴。
